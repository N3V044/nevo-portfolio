How It Works

Vulnerable - string formatting

User input is inserted directly into the SQL string. An attacker injects SQL to change the query logic.

# UNSAFE - input inside query string query = f"SELECT * FROM users WHERE username = '{username}' AND password = '{password}'" # With payload: ' OR '1'='1' -- SELECT * FROM users WHERE username = '' OR '1'='1' --' AND password = 'anything' # '1'='1' always TRUE # -- comments out password check # All rows returned - login bypassed

Secure - parameterized query

Input is passed as a parameter. The database treats it as data only.

# SAFE - ? placeholders cursor.execute( "SELECT * FROM users WHERE username = ? AND password = ?", (username, password) ) # DB looks for literal username: # ' OR '1'='1' -- # No such user exists # Login rejected
